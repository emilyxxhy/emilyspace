Case Study 03

Brewing Data
Intelligence.

Transforming 150,000+ scattered transactions into a high-performance retail strategy. A study in Power Query automation and dynamic dashboard architecture.

The Retail Blindspot

"How do we see the peak of our sales day across 150k rows of raw noise?"

The retailer was swimming in data but drowning in manual reporting. They lacked a real-time view of Average Transaction Value (ATV) and couldn't identify the specific promotional windows that actually drove revenue.

The BI Engine

  • ETL Tool Power Query
  • Data Model Power Pivot
  • Visualization Excel BI
  • Optimization DAX / Slicers

The Brewing Process.

01. Extract & Purify (ETL)

Imported raw CSVs using Power Query. Automated the cleaning of date-time strings and merged separate location tables into a unified 150K-row schema.

02. Calculating the "Flavor" (KPIs)

Developed DAX-based calculated columns to track 'Time-of-Day' segments, allowing the dashboard to slice revenue by Morning vs. Afternoon windows.

03. Designing the UI Experience

Constructed an interactive layout using Slicers and Pivot Charts. Prioritized a mobile-friendly layout for store managers to use on-site.

Data-Driven Sips

The critical discovery: the 2 PM peak was driven by low-margin cold drinks. By introducing a "Bundle Boost" (pairing these drinks with high-margin snacks) specifically in the 1 PM – 3 PM window, we directly influenced the Average Transaction Value.

Project Reflections.

Design Lesson

Proficiency isn't just about the data model; it's about UI/UX in Business Intelligence. Ensuring that Slicers and KPIs were intuitively placed for non-technical retail managers was the true project hurdle.

Scaleable Impact

This project proved that for small-to-mid businesses, Power Query is often the most cost-effective and practical BI solution, demonstrating high adaptability beyond just Python or SQL.
