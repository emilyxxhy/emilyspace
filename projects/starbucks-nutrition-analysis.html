Case Study 02

Nutritional
Architecture.

Uncovering the hidden dimensions of 160+ beverages. A deep dive into K-Means Clustering and PCA to build a personalized, healthier recommendation engine.

The User Problem

"How do customers make healthy choices in a menu of 160+ complex items?"

With thousands of combinations, nutritional transparency is often lost. I aimed to categorize beverages by their "Nutritional DNA" to suggest smarter swaps without sacrificing the user's taste preference.

The Clustering
Workflow.

01. Feature Standardization

Since Calories and Caffeine have vastly different scales, I applied StandardScaler to prevent the model from being biased toward high-magnitude features.

02. K-Means Optimal K

Identified K=5 as the optimal elbow point, segmenting the menu into logical groups like 'Low-Cal Espresso' vs. 'Indulgent Blended Treats'.

03. PCA for Visualization

Reduced 10+ dimensions down to 2 principal components to plot the clusters, explaining 85% of the variance in the menu's nutritional profile.

Interactive Deployment

The Healthy
Swap Engine.

The final deliverable was a **Streamlit application** where users can input their favorite drink and receive 3 "Smart Swaps." These swaps are calculated using Cosine Similarity within the user's specific drink cluster, ensuring the taste profile remains consistent while the calorie count drops.

Health Optimized

Project Reflections.

Scientific Lesson

PCA was the hero of this project. Without reducing dimensions, explaining why a drink belonged to a certain cluster would have been an abstract nightmare for a non-technical audience.

UX Intuition

Data is useless if it's trapped in a notebook. Deploying to Streamlit transformed a static clustering analysis into a tool that actually helps people make healthier breakfast decisions.
